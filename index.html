About this portfolio

This site collects live, clickable product specifications I've authored for the IDEAS experimentation platform. Each project is a self-contained PM artifact — problem statement, success metrics, user scenarios, designs, and risks — paired with an interactive prototype where stakeholders can experience the proposed UX rather than just read about it.

The artifacts here were drafted, prototyped, and refined with GitHub Copilot CLI in the loop end-to-end. They reflect how I work as an AI-powered Principal PM: faster from problem to convincing artifact, with higher density of decision-quality detail per page.

Product Spec

Guardrail Decision Framework

A deterministic safe / extend / not-safe verdict for every guardrail

A guardrails-only decision framework that turns raw scorecard confidence intervals and pre-experiment power into a deterministic per-guardrail verdict — safe, extend, or not-safe — and a single scorecard-level roll-up of Ship, Hold, or Stop. Each guardrail declares a direction (higher-is-better or lower-is-better) and a practical-significance threshold Δ; the framework runs a CI-based safety check (CI upper bound below +Δ for lower-is-better metrics; CI lower bound above −Δ otherwise), classifies powered vs. underpowered against MDE at 80% power, and treats underpowered guardrails as genuinely uninformative — closing the gap that today lets ~11% of experiments ship on signals that are statistically uninformative on harm. Multi-variant experiments are evaluated per T/C cell, every verdict carries a single-line reason, and any owner override of a not-safe ruling requires a written justification, mitigation, and follow-up captured in an auditable record. Replaces inconsistent team-by-team judgment with a single shared rule dictionary so the same scorecard yields the same verdict across PMs, DS, and leadership reviews.

Mini Spec

Auto-File Bug / IcM on Guardrail Violations

Closing the loop from not-safe verdict to FC ownership

A mini spec that wires the Guardrail Decision Framework's verdicts directly into Feature Crew accountability. On every scorecard cadence run, any not-safe or extend verdict files an ADO bug to the owning FC within 5 minutes; Sev≤2 guardrails (reliability, safety, compliance) additionally fire IcM to the FC's on-call. Routing and de-duplication use the Growth-orchestrator metadata so an orchestrated experiment family produces exactly one parent bug per regression cluster — not one per child flight. Closes the gap between detection and action: today, not-safe verdicts can sit unread in scorecards; with this, an auditable, severity-mapped artifact lands in the FC's queue and on the on-call's pager within minutes of detection.

Unauthenticated User Experimentation

MUID-based randomization infrastructure for pre-login A/B tests

An experimentation infrastructure spec that establishes MUID (Microsoft User ID) as the primary randomization unit for unauthenticated Copilot page experiments — replacing session-based randomization that suffers from severe carry-over effects and reduces statistical power from 48% to 28%. The spec covers privacy review requirements, ExP/Iris flighting integration, MUID-level metric aggregation (Days Active, Short-Term Retention, # Sessions), and known limitations like first-visit timing gaps and cookie churn. Enables the Copilot Growth team to run statistically valid experiments on pre-login experiences with consistent variant assignment across returning visitors.

Product Spec

Anytime-Valid Inference for Experiments

Continuous-monitoring safety detection for online A/B tests

An always-on safety layer for online A/B experimentation that lets release engineers, PMs, and data scientists watch a live experiment continuously without paying the classic peeking penalty — where naive repeated looks push false-positive rates from 5% to ~70%. Built on closed-form confidence sequences (Lindon, Ham, Tingley & Bojinov 2024) and a multinomial sequential SRM test (Lindon & Malek, NeurIPS 2022), it computes Type-I-controlled bounds at every sample size from the same regression tables an experimentation platform already produces. A near-real-time data path modeled on a NRT Service feeds a shrinking-band dashboard and ICM/Geneva alerts, turning every ramped flight into a continuously-monitored canary — targeting time-to-detect under 15 minutes (vs. ~14 hours today) and 40–50% sample-size savings, with mandatory 90-day shadow-mode validation per tenant.
